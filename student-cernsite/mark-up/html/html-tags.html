HTML Tags

This is a list of tags used in the HTML language. Each tag starts with a tag opener (<) and ends with a tag closer (>). Many tags have corresponding closing tags which are identical except for a slash after the tag opener (e.g., <TITLE>).

Some tags take parameters, called attributes. Attributes appear after the tag, separated by spaces, and affect rendering or behavior. For example, see HTML and elsewhere.

Title

The title of a document is given between <TITLE> tags. The text inside is the title for the hypertext node. There should only be one title per node. It may not contain anchors or paragraph marks. It may appear in history lists or window labels.

Next ID

Obsolete: Only used by NeXT Browser. This tag may be ignored. Example use: <NEXT ID=123>

Base Address

Anchors specify addresses of other documents. If a document is visible at multiple addresses, a base address may be used to resolve relative links. NOT CURRENTLY USED.

Anchors

Anchors define links in two forms: starting or destination of a link. Attributes include:

  • HREF: Start of a link to another document. Example: <a href="../markup.html">HTML</a>
  • NAME: Destination of a link. Example: <a name="anchor1">Destination</a>
  • TYPE: Relationship type. Optional. Default is void.

IsIndex

Marks a document as an index for keyword searches. Format: append ?keywords to the node address.

Plaintext

All following text is literal. No closing tag. Example:

This text is interpreted literally.
Line breaks and spacing are preserved.
< and > are treated as text, not tags.

Headings

HTML supports several levels of headings: <H1> to <H6>. H1 is recommended at the start of a node. Closing tags are required.

Paragraphs

Indicated with <P>. Marks the start of a new paragraph. Closing tags optional in early HTML.

Lists

Three main types:

  • UL: Unordered list
  • MENU: Compact list, one item per line
  • DIR: Short items, often arranged in columns

Glossaries

Definition lists using <DL>, with terms (<DT>) and definitions (<DD>). Example:

Term
A word or phrase being defined. See relative name.
Definition
The explanation of the term. See network address format.

Example Sections

Fixed-width text using <PRE> or <XMP>. Example:

12345678
! ! ! !
Column: First Second Third
Junk: The Quick brown fox jumps over the lazy dog
