This is a glossary of terms used within the ZackiSchool WorldWideWeb project. In most cases, their use corresponds to conventional use in hypertext circles.

Glossary of Hypertext Terms

Anchor
An area within the content of a node which is the source or destination of a link. Clicking the anchor follows the link, displaying the content at the opposite end. Anchors may highlight or use a special symbol. Also called "span", "region", "button", or "extent".
Annotation
Linking a new commentary node to an existing node. Readers providing feedback improves information quality. Annotation and multiuser collaboration
Authoring
The process of writing a document. Emphasizes that document creation involves more than just writing.
Back link
A link implied from an existing link in the opposite direction. See: Building back-links.
Browser
A program for reading hypertext, viewing node contents, and navigating links.
Button
An anchor that is the source of a link, often displayed as a push-button.
Card
An alternative term for a node in systems like HyperCard, limited to a single page.
Client
A program requesting services from another. Browsers are clients of servers.
Database
A collection of nodes. Links may be external or internal.
Daemon
A program running independently, performing tasks like building indexes and back-links. Servers under Unix are daemons.
Document
A node or collection of related nodes. Preferred term in W3 documentation.
Domain
A unit of protection, possibly corresponding to a database.
External
A link to a node in a different database.
Node
A unit of information, also called a frame or card. Distinct from network hosts.
Link
A relationship between two anchors, either internal or external.
Server
A program providing services to a client, typically serving hypertext content.
Web
A set of interconnected nodes via links. See also Topology.
